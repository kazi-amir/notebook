### 1. Summary

This PR introduces the hooks wrapper infrastructure: a `HookPackageConfig` class that parses and validates `Config.json` against a JSON schema, and an `IacHookWrapper` that discovers hook packages and dispatches the `prepare` functionality via dynamic module import. The overall architecture is sound, but there are two blocking correctness issues: the `entry_method` JSON schema definition omits `required` constraints on `module` and `method`, which allows a structurally incomplete config to pass validation and cause an unhandled `KeyError` at runtime; and `_execute_as_module` blindly casts its return value to `Dict` without runtime validation, causing an `AttributeError` if a hook method returns `None` instead of a dict.

### 2. Blockers

**Blocker 1: `entry_method` JSON schema is missing `"required": ["module", "method"]`**

- **What's wrong:** In `hook_config_schema.json`, the `entry_method` definition lists `module` and `method` as `properties` but does not declare them in a `"required"` array. This means a config with `"entry_method": {}` passes `jsonschema.validate()` successfully.
- **Why it matters:** After validation, `HookPackageConfig.__init__` converts each functionality dict into a `HookFunctionality(func_dict["entry_method"])` NamedTuple. The properties `functionality.module` and `functionality.method` are implemented as `self.entry_method["module"]` and `self.entry_method["method"]`. If `entry_method` is an empty dict, both of these raise an unhandled `KeyError` rather than a clean `InvalidHookPackageConfigException` or `InvalidHookWrapperException`. The schema is the contract that is supposed to prevent malformed configs from reaching the execution path.
- **Exact fix:** Add `"required": ["module", "method"]` to the `entry_method` definition in `hook_config_schema.json` so that any config missing either field is rejected at validation time with a proper `InvalidHookPackageConfigException`.
- **Where:** `samcli/lib/hook/hook_config_schema.json`, the `"entry_method"` definition under `"$defs"`.

---

**Blocker 2: `_execute_as_module` does not validate that the hook method returns a `Dict`**

- **What's wrong:** In `hook_wrapper.py`, `_execute_as_module` ends with `result = getattr(mod, method)(params)` followed by `return cast(Dict, result)`. In Python, `cast` is a static type hint only — it performs no runtime check or conversion. If the hook method returns `None` or any non-dict value, `result` is still `None` (or whatever the method returned).
- **Why it matters:** The caller `_execute` returns this value directly to `prepare()`, which then calls `output.get("IACApplications", {})`. If `output` is `None`, this raises `AttributeError: 'NoneType' object has no attribute 'get'` — an unhandled exception with no meaningful error message. Users of the hook see a raw Python traceback instead of a clear `InvalidHookWrapperException`.
- **Exact fix:** After `result = getattr(mod, method)(params)`, add a runtime check such as `if not isinstance(result, dict): raise InvalidHookWrapperException(...)` before returning, so that non-dict returns are caught immediately with an actionable error message.
- **Where:** `samcli/lib/hook/hook_wrapper.py`, inside the `_execute_as_module` function, after calling `getattr(mod, method)(params)`.

### 3. Non-blocking suggestions

- **Unused imports in `test_hook_wrapper.py`:** The file imports `from asyncio import subprocess` and `from sys import stderr, stdin`, none of which are used anywhere in the test file. These should be removed to keep the test module clean.
- **`jsonschema` property re-reads from disk on every call:** The `jsonschema` property in `HookPackageConfig` opens and parses the schema JSON file each time it is accessed. Since it is called once during `__init__`, this is not a performance problem in practice, but using a class-level cached attribute (e.g., `@functools.cached_property` or a class variable loaded once) would be more robust and avoids unnecessary I/O if the property is ever called more than once.
