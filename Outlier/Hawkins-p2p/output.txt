Currently, the `ComClassGenerator` in the codebase violates the separation of concerns by handling both the generation of COM class wrappers and the reporting of diagnostics (e.g., checking for missing `partial` modifiers or disabled unsafe code). This mixes code emission logic with diagnostic logic, complicating the generator's plumbing and relying on `DiagnosticOr<T>` wrappers. The diagnostic reporting responsibilities need to be extracted out of the `ComClassGenerator` and moved into a dedicated Roslyn `DiagnosticAnalyzer` to simplify the generator pipeline and improve maintainability.

## Context
The `ComClassGenerator` source generator in `System.Runtime.InteropServices` currently handles both the generation of COM class code and the reporting of related diagnostics (such as checking if `AllowUnsafeBlocks` is enabled or if the class has a `partial` modifier). ## Problem Statement
We need to refactor the COM class generation process to strictly adhere to the separation of concerns. Diagnostic reporting should be entirely extracted out of `ComClassGenerator` and moved into a dedicated Roslyn `DiagnosticAnalyzer`. The generator should be simplified to focus solely on code emission.

### Parsing Script (`parsing.py`)
```python
import sys
import json
import re
from typing import List, Dict

def parse_test_output(stdout_content: str, stderr_content: str) -> List[Dict[str, str]]:
    tests = []
    # Match standard dotnet test detailed output for Passed/Failed tests
    # Example: "Passed ComInterfaceGenerator.Unit.Tests.ComClassGeneratorDiagnostics.UnsafeCodeNotEnabledWarns"
    pattern = re.compile(r"^\s*(Passed|Failed|Skipped)\s+(.+)$", re.MULTILINE)
    
    for match in pattern.finditer(stdout_content):
        status_raw = match.group(1).upper()
        test_name = match.group(2).strip()
        
        # Normalize status
        if status_raw == "PASSED":
            status = "PASSED"
        elif status_raw == "FAILED":
            status = "FAILED"
        elif status_raw == "SKIPPED":
            status = "SKIPPED"
        else:
            status = "ERROR"
            
        tests.append({
            "name": test_name,
            "status": status
        })
        
    return tests

if __name__ == "__main__":
    if len(sys.argv) != 4:
        print("Usage: python3 parsing.py <stdout.txt> <stderr.txt> <results.json>")
        sys.exit(1)

    with open(sys.argv[1], 'r') as f:
        stdout = f.read()
    with open(sys.argv[2], 'r') as f:
        stderr = f.read()

    parsed_tests = parse_test_output(stdout, stderr)

    with open(sys.argv[3], 'w') as f:
        json.dump({"tests": parsed_tests}, f, indent=4)
```
